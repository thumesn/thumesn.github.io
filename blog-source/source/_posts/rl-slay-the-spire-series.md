这个专题记录一个长期实验：用强化学习去训练一个能玩《杀戮尖塔》的 agent。它不会从一开始就追求“通关”，而是先把问题拆小：环境怎么建模，动作空间怎么压缩，reward 怎么设计，baseline 怎么评估，再逐步推进到策略学习和搜索结合。

我更想把它写成一组可复盘的工程记录，而不是一篇一次性教程。强化学习项目最容易卡住的地方，往往不是 PPO、DQN 或 Actor-Critic 的公式，而是环境接口、状态表示、非法动作、随机性和评估方式。

## 为什么选择杀戮尖塔

《杀戮尖塔》适合拿来做 RL 实验，因为它同时具备几个有意思的特征：

1. **状态空间大**：卡组、遗物、血量、敌人意图、药水、路线都会影响决策。
2. **动作有强约束**：战斗中不是所有牌都能打，地图上不是所有路线都能走。
3. **长期收益明显**：一张牌当前很强，不代表它适合后续卡组。
4. **随机性很强**：抽牌、奖励、事件、敌人组合都会改变策略价值。

这些特征让它比 CartPole 这类入门环境复杂很多，但又比完全开放世界更可控。对学习 RL 来说，它是一个足够折磨、也足够具体的中间项目。

## 专题路线

我准备按下面的顺序写：

### 1. 环境建模

第一步是把游戏拆成 RL 能理解的问题：什么是 state，什么是 action，什么时候 episode 结束，以及哪些信息应该进入 observation。

- 当前文章：[环境建模：先把游戏拆成状态、动作和奖励](/2026/07/20/rl-slay-the-spire-environment-modeling/)

### 2. 动作空间压缩

原始动作空间会非常麻烦。战斗时可以出牌、选择目标、结束回合；奖励时可以选牌、跳过、拿遗物；地图时还要选路线。后续需要把这些动作统一成一个可管理的接口。

我会重点记录三件事：

1. 如何表达非法动作 mask
2. 如何把组合动作拆成多步决策
3. 如何避免 agent 学到“卡死流程”的行为

### 3. Reward 设计

如果只在通关时给 reward，训练会非常稀疏。如果每一步都给太多 shaping reward，agent 又可能学到短视行为。这个专题会从最粗的胜负 reward 开始，再逐步加入血量、战斗损失、卡组质量等中间信号。

### 4. Baseline 与评估

在训练复杂 agent 前，需要先写一些简单 baseline：

1. 随机合法动作
2. 简单启发式策略
3. 固定选牌规则

没有 baseline，训练曲线很容易自我感动。一个模型看起来 reward 上升，不代表它真的学会了游戏，也可能只是避开了某些会扣分的动作。

### 5. 策略学习

等环境和评估稳定后，再考虑算法。初期我会优先试 PPO，因为它适合离散动作和复杂策略；后面再看能否加入 value-based 方法、模仿学习或搜索。

### 6. 搜索与规划

杀戮尖塔的很多决策带有规划性质。比如当前战斗怎么打，路线怎么选，卡组未来怎么成型。纯 RL 可能不够稳，后续可以考虑把策略网络和搜索结合起来。

## 这个专题会记录什么

我会重点记录可复现的东西：

- 环境接口怎么设计
- observation 和 action 怎么编码
- reward 为什么这么设
- 训练曲线怎么看
- 哪些实验失败了
- 哪些 bug 会让模型学歪

如果只写成功结果，这个专题就没有太大价值。RL 项目的真实经验通常藏在失败里：reward 设错、动作 mask 漏了、评估脚本不一致、训练随机种子导致结论反复变化。

## 当前判断

这个项目的核心难点不是“用哪个算法”，而是先把游戏变成一个稳定、可评估、可迭代的学习环境。算法可以换，但环境抽象一旦混乱，后续实验都会变成噪声。

所以第一阶段目标很明确：先做一个小而硬的环境接口，让 agent 能在受限场景里完成合法决策。等这个地基稳定，再谈完整爬塔。
