这篇文章整理 MoE 中最核心的 token 路由问题：每个 token 如何选择专家，以及这件事为什么直接决定模型效果和推理成本。

## MoE 的核心思想

MoE（Mixture of Experts）不是让所有参数都参与每次计算，而是让每个 token 只激活少量专家（例如 top-1 或 top-2）。

这样可以在总参数规模很大的同时，把单次前向计算量控制在可接受范围内。

## Token 路由在做什么

路由器会对当前 token 输出一组专家得分，然后选择得分最高的若干专家参与计算。

这个过程的关键点有两个：

1. 选谁（路由质量）
2. 负载是否均衡（专家利用率）

## 常见工程挑战

- 热门专家过载：少数专家被大量 token 选择
- 冷门专家闲置：训练利用率低，容量被浪费
- 路由抖动：训练早期不稳定，影响收敛

## 常见解决思路

1. 加入负载均衡相关损失，抑制专家极端倾斜。
2. 设定专家容量上限，避免单个专家被打爆。
3. 调整 top-k、温度和路由噪声，平衡探索与稳定。

## 实战建议

1. 先看路由统计再谈模型好坏：每层专家命中分布、溢出率、丢弃率。
2. 线上压测要覆盖长上下文和高并发场景，路由问题常在高压下暴露。
3. 把路由日志纳入常规监控，不要只盯最终基准分数。

## 小结

MoE 的收益不只来自“参数更多”，更来自“路由是否稳定且均衡”。token 路由做不好，MoE 很容易从优势变成负担。
