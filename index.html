  • Attention 全览：从 MHA 到 GQA、RoPE 与工程优化

    2026-03-03

    这篇文章作为 Attention 系列总览，目标是用一条主线串起常见机制：基础计算、头部变体、位置编码、缓存与工程加速。

    1. 注意力在算什么

    自注意力的核心是让每个 token 根据上下文动态聚合信息。标准形式是 Scaled Dot-Product Attention,原始论文来自《Attention Is All You Need》（Vaswani et al., 2017）

    • 先把输入投影到 Q、K、V
    • 用 Q 和 K 算相关性分数
    • 对分数做 softmax 后加权求和 V

    什么是注意力？

    注意力机制让模型在处理输入时能够动态地关注不同部分的信息。它通过计算输入序列中各个元素之间的相关性来决定哪些信息更重要，从而提高模型的表达能力和性能。简单理解,输出中存在重要信息时，注意力机制会让模型更关注这些信息，从而更好地理解和生成文本。

    • Query（查询）：表示当前 token 的查询向量，用于计算与其他 token 的相关性。举个生活化的例子，假设你在阅读一篇文章，Query 就像是你当前关注的内容或问题。
    • Key（键）：表示输入序列中每个 token 的键向量，用于与 Query 计算相关性。Key 就像是文章中每个词的标签或特征。
    • Value（值）：表示输入序列中每个 token 的值向量，用于根据相关性加权求和得到最终输出。Value 就像是文章中每个词的实际内容或信息。

    举个生活化的例子,Q 就是你关注的问题，比如“这篇文章的主要观点是什么？”，K 是文章中每个词的标签或特征，比如“观点”、“细节”、“例子”等，V 是文章中每个词的实际内容或信息。当你阅读文章时，你会根据你的关注点（Query）来寻找相关的信息（Key），并根据相关性来获取有用的信息（Value）。注意力机制就是在模型内部模拟这个过程，让模型能够更好地理解和生成文本。

    公式

    $$
    \text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
    $$

    其中 $d_k$ 是 Key 向量的维度，$\sqrt{d_k}$ 是缩放因子，防止相关性分数过大导致 softmax 梯度消失。

    为什么要缩放？
    因为 Q 和 K 的维度通常较大，点积结果可能会有较大的值，导致 softmax 输出过于集中，梯度消失。缩放后可以保持数值稳定性。方差过大会导致 softmax 输出过于集中，模型难以学习到有效的梯度。通过除以 $\sqrt{d_k}$，可以使得点积的值保持在一个合理的范围内，从而避免梯度消失问题。
    为什么除以 $\sqrt{d_k}$？
    因为点积的期望值与维度成正比： 证明：假设 Q 和 K 的元素独立同分布，均值为 0，方差为 1，那么点积 $QK^T$ 的期望值为 $d_k$，因此需要除以 $\sqrt{d_k}$ 来保持数值稳定。
    为什么不存在 某一个 Q 的值特别大，比如某一个维度上有一个值特别大？因为 Q 和 K 的元素通常是通过线性变换得到的，且在训练过程中会有正则化和梯度更新来防止某个维度的值过大。此外，缩放因子 $\sqrt{d_k}$ 也有助于缓解这个问题。
    Q,K 是如何得到的？它们是通过输入序列的线性变换得到的。具体来说，输入序列中的每个 token 都会被映射到一个向量空间中，然后通过三个不同的线性层分别生成 Q、K 和 V 向量。这些线性层的权重是在训练过程中学习得到的，因此模型可以根据任务需求自动调整 Q、K 和 V 的表示方式.

    什么是自注意力

    自注意力（Self-Attention）是指在计算注意力时，Query、Key 和 Value 都来自同一个输入序列。这使得模型能够在处理每个 token 时同时考虑整个序列的信息，从而捕捉长距离依赖关系。

    举个例子，这样的一句话“我喜欢吃苹果，因为它很甜。”，在计算自注意力时，每个词都会与其他词进行交互。例如，它可以让模型理解“它”指代的是“苹果”，从而更好地理解句子的含义。

    • 从零开始计算自注意力：
      • 输入序列 [“I”, “like”, “to”, “eat”, “apples”]
      • tokenizer 将每个词转换为向量表示，例如：
        • “I” -> [0.1, 0.2, 0.3]
        • “like” -> [0.4, 0.5, 0.6]
        • “to” -> [0.7, 0.8, 0.9]
        • “eat” -> [1.0, 1.1, 1.2]
        • “apples” -> [1.3, 1.4, 1.5]
      • 通过线性变换生成 Q、K、V 向量，例如：
        • $Q_{“I”} $ = [0.1, 0.2, 0.3] * W^Q$
        • $K_{“I”} $ = [0.1, 0.2, 0.3] * W^K$
        • $V_{“I”} $ = [0.1, 0.2, 0.3] * W^V$
        • 其他词同理。
        • 各自的shape:
          • Q: (seq_len, d_model) -> (seq_len, d_k)
          • K: (seq_len, d_model) -> (seq_len, d_k)
          • V: (seq_len, d_model) -> (seq_len, d_v)
        • 常见的模型的比如gpt-3.5-turbo中，d_model=4096, d_k=d_v=1024。
      • 计算相关性分数，例如：
        • $score_{“I”, “like”} = Q_{“I”} \cdot K_{“like”}^T / \sqrt{d_k}$
        • 其他词同理。
      • 对分数做 softmax 后加权求和 V，例如：
        • $Attention_{“I”} = \sum_{j} \text{softmax}(score_{“I”, j}) \cdot V_j$
        • 其他词同理。
        • 最终如果某个词和其他词的相关性较高，那么它在输出中就会占更大的权重，从而让模型更关注这个词的信息。

    2. 从单头到多头：MHA

    Multi-Head Attention（MHA）把注意力分成多个头并行计算。简单来说，我们会认为每一个词有多个“视角”来关注其他词的信息，每个头负责一个视角。这样可以让模型捕捉到更多样化的关系。

    代价也很直接：头越多，计算与显存开销越大，推理阶段 KV Cache 压力更明显。

    3. 头部变体：MQA 与 GQA

    为降低推理成本，常见两种变体：

    1. MQA：所有 Query 头共享同一组 Key/Value，极省 KV 开销。
    2. GQA：多个 Query 头共享一组 Key/Value，在质量与成本间折中。

    详细拆解见：

    4. 位置编码：为什么常用 RoPE

    注意力机制本身不感知顺序，所以需要位置编码。RoPE 通过旋转变换把位置信息注入向量，在长上下文中表现通常更稳。

    详细拆解见：

    • RoPE（旋转位置编码）理解与应用

    5. 实现细节：QKV Bias

    QKV 线性层中的 bias 看起来很小，但会影响训练动态与收敛行为，是否启用应通过任务实验决定。

    详细拆解见：

    • QKV Bias 在 Transformer 中的作用

    6. 推理侧重点：KV Cache

    自回归推理时，历史 token 的 K/V 会缓存起来避免重复计算。上下文变长后，KV Cache 往往成为显存与吞吐瓶颈，这也是 MQA/GQA 受欢迎的重要原因。

    评估时建议同时看：

    • 吞吐（tokens/s）
    • 延迟（首 token 与平均 token）
    • 长上下文稳定性

    7. 工程加速：FlashAttention（概念）

    FlashAttention 的核心价值是优化注意力计算的访存路径，减少不必要的内存读写，从而提升速度并降低显存压力。

    它本质是“算子工程优化”，不改变注意力机制定义，但显著改变实际运行效率。

    8. 怎么读 Attention 相关论文

    我的建议是固定三问：

    1. 这项改动改善了什么瓶颈（质量、显存、延迟、吞吐）？
    2. 代价是什么（训练复杂度、部署复杂度、稳定性风险）？
    3. 在我的目标任务和预算下是否划算？

    小结

    Attention 不是单一技术点，而是一组“机制 + 工程”组合。先用总览建立坐标，再按专题深入，学习和落地都会更快。

  • PPO 在 LLM 对齐中的作用：RLHF 经典方案速记

    2026-03-03

    PPO（Proximal Policy Optimization）是 RLHF 里最经典的策略优化方法之一。它的目标是在提升回答偏好的同时，控制策略更新不要过激。

    在 RLHF 链路中的位置

    典型流程是：

    1. 先做 SFT，得到可用初始策略
    2. 用偏好数据训练奖励模型
    3. 用 PPO 优化策略模型，提升高奖励回答概率

    PPO 的核心价值

    PPO 通过约束每次更新幅度，让训练更稳定，减少策略突然崩坏的风险。

    对大模型来说，这种“稳步更新”非常关键。

    成本与难点

    • 流程更长，组件更多（策略模型、奖励模型、参考模型）
    • 超参数敏感，调参成本高
    • 工程复杂度与训练资源要求较高

    什么时候用 PPO

    当你有较完整的 RLHF 基建，且追求更精细的行为控制时，PPO 仍然是值得投入的方案。

    如果团队资源有限，很多场景会先用 DPO 获得更高性价比。

    小结

    PPO 不是“过时方案”，而是“成本更高但可控性更强”的经典工具。要不要用，核心看团队资源与目标精度。

  • DPO（直接偏好优化）笔记：不用奖励模型也能做偏好对齐

    2026-03-03

    DPO（Direct Preference Optimization）是一种常见的偏好对齐方法。它直接利用“好回答 vs 差回答”的成对数据优化模型，无需单独训练奖励模型。

    DPO 的直觉

    给定同一提示词下的两条回答，DPO 让模型提高优选回答概率，压低劣选回答概率。

    相比传统 RLHF 流程，DPO 更直接，训练链路更短。

    为什么流行

    1. 训练实现相对简单
    2. 稳定性通常较好
    3. 对工程资源要求更友好

    关键前提

    DPO 的效果非常依赖偏好数据质量。如果偏好标注噪声大，模型会学到相互矛盾的信号。

    常见实践要点

    • 参考模型（reference model）选择会影响训练行为
    • 偏好数据要覆盖你真正关心的任务域
    • 线上验证要看“有用性 + 安全性 + 风格一致性”

    小结

    DPO 的价值在于把偏好对齐流程“去复杂化”。在很多团队里，它是从 SFT 迈向高质量助手的高性价比路径。

  • SFT（监督微调）笔记：把基础模型变成可用助手的第一步

    2026-03-03

    SFT（Supervised Fine-Tuning）通常是对齐阶段的第一步：用高质量指令数据把基础模型从“会续写”拉到“会按指令回答”。

    SFT 解决的核心问题

    预训练模型有知识和语言能力，但不一定稳定遵循指令格式。SFT 的目标是把目标行为直接示范给模型。

    数据决定上限

    SFT 的效果高度依赖数据：

    1. 指令覆盖面是否足够广
    2. 回答质量是否一致
    3. 格式规范是否稳定

    如果示例数据本身不稳，模型只会学到不稳的习惯。

    常见收益

    • 指令跟随能力明显提升
    • 回答结构更可控
    • 后续偏好优化（DPO/PPO）更容易收敛

    常见坑

    • 数据同质化严重，泛化变差
    • 过度模板化，回答僵硬
    • 只看 loss 下降，忽视真实任务表现

    实战建议

    1. 先做一版小而干净的数据集，建立稳定基线。
    2. 增量扩数据时做分桶评估，避免“越训越差”。
    3. SFT 后马上做任务集验证，再决定是否进入 DPO/PPO。

    小结

    SFT 不是终点，但往往决定后续对齐能否顺利推进。把数据质量打牢，比盲目堆样本更重要。

  • MoE Token 路由机制笔记：为什么能在成本内提升能力

    2026-03-03

    这篇文章整理 MoE 中最核心的 token 路由问题：每个 token 如何选择专家，以及这件事为什么直接决定模型效果和推理成本。

    MoE 的核心思想

    MoE（Mixture of Experts）不是让所有参数都参与每次计算，而是让每个 token 只激活少量专家（例如 top-1 或 top-2）。

    这样可以在总参数规模很大的同时，把单次前向计算量控制在可接受范围内。

    Token 路由在做什么

    路由器会对当前 token 输出一组专家得分，然后选择得分最高的若干专家参与计算。

    这个过程的关键点有两个：

    1. 选谁（路由质量）
    2. 负载是否均衡（专家利用率）

    常见工程挑战

    • 热门专家过载：少数专家被大量 token 选择
    • 冷门专家闲置：训练利用率低，容量被浪费
    • 路由抖动：训练早期不稳定，影响收敛

    常见解决思路

    1. 加入负载均衡相关损失，抑制专家极端倾斜。
    2. 设定专家容量上限，避免单个专家被打爆。
    3. 调整 top-k、温度和路由噪声，平衡探索与稳定。

    实战建议

    1. 先看路由统计再谈模型好坏：每层专家命中分布、溢出率、丢弃率。
    2. 线上压测要覆盖长上下文和高并发场景，路由问题常在高压下暴露。
    3. 把路由日志纳入常规监控，不要只盯最终基准分数。

    小结

    MoE 的收益不只来自“参数更多”，更来自“路由是否稳定且均衡”。token 路由做不好，MoE 很容易从优势变成负担。

  • RMSNorm 笔记：在大模型中为什么常替代 LayerNorm

    2026-03-03

    这篇笔记聚焦 RMSNorm 的核心差异和工程意义。

    RMSNorm 与 LayerNorm 的区别

    LayerNorm 会做均值中心化 + 方差归一化；RMSNorm 主要使用均方根尺度进行归一化，结构更简洁。

    在大模型中的常见优势

    1. 计算更简洁，工程实现成本低
    2. 训练稳定性通常不错
    3. 在大规模训练中已被大量实践验证

    什么时候需要谨慎

    • 迁移旧模型时，归一化策略变化可能影响原有超参数最优点
    • 不同任务对归一化的敏感度不同

    实战建议

    1. 归一化层替换后，重新做学习率和 warmup 调参。
    2. 重点关注训练前期震荡和后期收敛平台。
    3. 和激活函数、残差比例一起联动评估。

    小结

    RMSNorm 的价值在于“更简单且足够有效”，这使它在现代 LLM 结构中非常常见。

  • QKV Bias 在 Transformer 中的作用

    2026-03-03

    这篇文章记录 QKV Bias 这个容易被忽略、但会影响训练行为的细节。

    QKV Bias 是什么

    在注意力层里，输入会投影成 Q、K、V。QKV Bias 指的是这些线性投影中的偏置项。

    为什么值得关注

    • 会影响训练初期动态
    • 可能影响收敛速度与稳定性
    • 在不同模型规模和训练策略下，收益并不总是同向

    是否一定要开

    不是。是否启用 QKV Bias 应该由实验决定：

    1. 任务类型
    2. 模型规模
    3. 数据规模与分布
    4. 训练超参数设置

    建议的评估方式

    做两组最小对照实验：

    • 开启 QKV Bias
    • 关闭 QKV Bias

    同时比较：

    • 收敛曲线
    • 验证集效果
    • 推理效率与稳定性

    小结

    QKV Bias 是“看起来小、但可能有影响”的结构开关。不要凭经验拍板，尽量用小规模可复现实验做决策。

  • RoPE（旋转位置编码）理解与应用

    2026-03-03

    这篇笔记整理 RoPE 的核心思路，以及它为什么适合现代大模型。

    问题背景

    注意力机制本身不感知顺序，需要位置编码告诉模型“谁在前、谁在后”。

    RoPE 的核心思路

    RoPE 通过对向量做“旋转变换”注入位置信息，使相对位置信息自然体现在注意力计算中。

    相比只加绝对位置向量，RoPE 在长序列建模里往往更稳。

    为什么它常见

    1. 与自注意力计算形式契合
    2. 对相对位置信息建模更自然
    3. 在长上下文扩展场景中表现普遍较好

    实战注意点

    • 长上下文外推能力不只由 RoPE 决定，还受训练数据和策略影响。
    • 做上下文扩展时，常要配合额外缩放或插值策略。

    小结

    RoPE 是当前 LLM 里很主流的位置编码方式，优势在于“结构简单 + 长序列友好”。
