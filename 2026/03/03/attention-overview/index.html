Attention 全览：从 MHA 到 GQA、RoPE 与工程优化

这篇文章作为 Attention 系列总览，目标是用一条主线串起常见机制：基础计算、头部变体、位置编码、缓存与工程加速。

1. 注意力在算什么

自注意力的核心是让每个 token 根据上下文动态聚合信息。标准形式是 Scaled Dot-Product Attention,原始论文来自《Attention Is All You Need》（Vaswani et al., 2017）

  • 先把输入投影到 Q、K、V
  • 用 Q 和 K 算相关性分数
  • 对分数做 softmax 后加权求和 V

什么是注意力？

注意力机制让模型在处理输入时能够动态地关注不同部分的信息。它通过计算输入序列中各个元素之间的相关性来决定哪些信息更重要，从而提高模型的表达能力和性能。简单理解,输出中存在重要信息时，注意力机制会让模型更关注这些信息，从而更好地理解和生成文本。

  • Query（查询）：表示当前 token 的查询向量，用于计算与其他 token 的相关性。举个生活化的例子，假设你在阅读一篇文章，Query 就像是你当前关注的内容或问题。
  • Key（键）：表示输入序列中每个 token 的键向量，用于与 Query 计算相关性。Key 就像是文章中每个词的标签或特征。
  • Value（值）：表示输入序列中每个 token 的值向量，用于根据相关性加权求和得到最终输出。Value 就像是文章中每个词的实际内容或信息。

举个生活化的例子,Q 就是你关注的问题，比如“这篇文章的主要观点是什么？”，K 是文章中每个词的标签或特征，比如“观点”、“细节”、“例子”等，V 是文章中每个词的实际内容或信息。当你阅读文章时，你会根据你的关注点（Query）来寻找相关的信息（Key），并根据相关性来获取有用的信息（Value）。注意力机制就是在模型内部模拟这个过程，让模型能够更好地理解和生成文本。

公式

$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

其中 $d_k$ 是 Key 向量的维度，$\sqrt{d_k}$ 是缩放因子，防止相关性分数过大导致 softmax 梯度消失。

为什么要缩放？
因为 Q 和 K 的维度通常较大，点积结果可能会有较大的值，导致 softmax 输出过于集中，梯度消失。缩放后可以保持数值稳定性。方差过大会导致 softmax 输出过于集中，模型难以学习到有效的梯度。通过除以 $\sqrt{d_k}$，可以使得点积的值保持在一个合理的范围内，从而避免梯度消失问题。
为什么除以 $\sqrt{d_k}$？
因为点积的期望值与维度成正比： 证明：假设 Q 和 K 的元素独立同分布，均值为 0，方差为 1，那么点积 $QK^T$ 的期望值为 $d_k$，因此需要除以 $\sqrt{d_k}$ 来保持数值稳定。
为什么不存在 某一个 Q 的值特别大，比如某一个维度上有一个值特别大？因为 Q 和 K 的元素通常是通过线性变换得到的，且在训练过程中会有正则化和梯度更新来防止某个维度的值过大。此外，缩放因子 $\sqrt{d_k}$ 也有助于缓解这个问题。
Q,K 是如何得到的？它们是通过输入序列的线性变换得到的。具体来说，输入序列中的每个 token 都会被映射到一个向量空间中，然后通过三个不同的线性层分别生成 Q、K 和 V 向量。这些线性层的权重是在训练过程中学习得到的，因此模型可以根据任务需求自动调整 Q、K 和 V 的表示方式.

什么是自注意力

自注意力（Self-Attention）是指在计算注意力时，Query、Key 和 Value 都来自同一个输入序列。这使得模型能够在处理每个 token 时同时考虑整个序列的信息，从而捕捉长距离依赖关系。

举个例子，这样的一句话“我喜欢吃苹果，因为它很甜。”，在计算自注意力时，每个词都会与其他词进行交互。例如，它可以让模型理解“它”指代的是“苹果”，从而更好地理解句子的含义。

  • 从零开始计算自注意力：
    • 输入序列 [“I”, “like”, “to”, “eat”, “apples”]
    • tokenizer 将每个词转换为向量表示，例如：
      • “I” -> [0.1, 0.2, 0.3]
      • “like” -> [0.4, 0.5, 0.6]
      • “to” -> [0.7, 0.8, 0.9]
      • “eat” -> [1.0, 1.1, 1.2]
      • “apples” -> [1.3, 1.4, 1.5]
    • 通过线性变换生成 Q、K、V 向量，例如：
      • $Q_{\text{I}} = [0.1, 0.2, 0.3] W^Q$
      • $K_{\text{I}} = [0.1, 0.2, 0.3] W^K$
      • $V_{\text{I}} = [0.1, 0.2, 0.3] W^V$
      • 其他词同理。
      • 各自的shape:
        • Q: (seq_len, d_model) -> (seq_len, d_k)
        • K: (seq_len, d_model) -> (seq_len, d_k)
        • V: (seq_len, d_model) -> (seq_len, d_v)
      • 常见的模型的比如gpt-3.5-turbo中，d_model=4096, d_k=d_v=1024。
    • 计算相关性分数，例如：
      • $score_{\text{I},\text{like}} = Q_{\text{I}} \cdot K_{\text{like}}^T / \sqrt{d_k}$
      • 其他词同理。
    • 对分数做 softmax 后加权求和 V，例如：
      • $Attention_{\text{I}} = \sum_{j} \text{softmax}(score_{\text{I}, j}) \cdot V_j$
      • 其他词同理。
      • 最终如果某个词和其他词的相关性较高，那么它在输出中就会占更大的权重，从而让模型更关注这个词的信息。

2. 从单头到多头：MHA

Multi-Head Attention（MHA）把注意力分成多个头并行计算。简单来说，我们会认为每一个词有多个“视角”来关注其他词的信息，每个头负责一个视角。这样可以让模型捕捉到更多样化的关系。

如何实现 MHA：

  • 将输入序列通过不同的线性层生成多个 Q、K、V 组，例如：
    • $Q^1 = X W^{Q_1}$, $K^1 = X W^{K_1}$, $V^1 = X W^{V_1}$
    • $Q^2 = X W^{Q_2}$, $K^2 = X W^{K_2}$, $V^2 = X W^{V_2}$
    • …
  • 每个头独立计算注意力，例如：
    • $Attention^1 = \text{Attention}(Q^1, K^1, V^1)$
    • $Attention^2 = \text{Attention}(Q^2, K^2, V^2)$
    • …
  • 最后将所有头的输出拼接起来并通过线性层得到最终输出，例如：
  • $Output = \text{Concat}(Attention^1, Attention^2, …)) W^O$
  • 各自的shape:
    • Attention^i: (seq_len, d_v)
    • Concat(Attention^1, Attention^2, …): (seq_len, num_heads * d_v)
    • Output: (seq_len, d_model)
      所以有 $d_model = num_heads * d_v$。

MHA 有多费显存？

简单直接的回答是：在保持模型总维度（$d_{model}$）不变的前提下，多头注意力（MHA）相比单头注意力（SHA）在理论计算量（FLOPs）和参数量上几乎是一样的，但在实际工程中会有微小的额外显存和计算开销。

1. 前提：公平比较的标准

为了公平比较，我们通常假设模型的总表示维度 $d_{model}$ 保持不变。

  • 单头注意力 (SHA)：1 个头，头的维度 $d_k = d_{model}$。
  • 多头注意力 (MHA)：$h$ 个头，每个头的维度 $d_k = d_{model} / h$。

2. 计算时间花费 (Computation Time / FLOPs)

理论计算量 (FLOPs)：基本相同
注意力机制的核心计算是 $Q \times K^T$ 和 $\text{Score} \times V$。

  • MHA: $h \times (序列长度^2 \times d_k) = h \times (L^2 \times \frac{d_{model}}{h}) = L^2 \times d_{model}$
  • SHA: $1 \times (序列长度^2 \times d_{model}) = L^2 \times d_{model}$
    结论：理论上浮点运算次数完全一致。

实际计算时间：MHA 有微小开销，但可能更快

  • 额外开销：MHA 需要对张量进行 split（切分）、transpose（转置以交换头和序列维度）、concat（拼接）等操作。这些操作本身不产生大量计算，但会消耗少量的 GPU 内核启动时间和内存带宽。
  • 并行优势：GPU 擅长并行计算。MHA 将一个大矩阵乘法拆分成 $h$ 个小矩阵乘法，可以更好地利用 GPU 的流多处理器（SM）。在某些硬件配置下，MHA 甚至可能比单个巨大的矩阵乘法（SHA）效率更高，因为小矩阵更容易填满缓存。
  • 总结：时间开销差异通常在 1% - 5% 以内，往往可以忽略不计。

3. 显存花费 (VRAM)

参数量 (Weights)：完全相同

  • MHA: $h$ 组 $W_Q, W_K, W_V, W_O$，每组大小是 $(d_{model} \times \frac{d_{model}}{h})$。总参数量 $\approx 4 \times d_{model}^2$。
  • SHA: 1 组 $W_Q, W_K, W_V, W_O$，大小是 $(d_{model} \times d_{model})$。总参数量 $\approx 4 \times d_{model}^2$。
    结论：模型权重占用的显存是一样的。

激活显存 (Activations)：MHA 略高
这是两者显存差异的主要来源，主要在于注意力分数矩阵 (Attention Scores) 的存储。

  • 在反向传播时，需要保存 Softmax 后的注意力矩阵用于计算梯度。
  • SHA: 保存 1 个 $L \times L$ 的矩阵。
  • MHA: 保存 $h$ 个 $L \times L$ 的矩阵（每个头一个）。
  • 差异：MHA 需要多存储 $(h-1) \times L^2$ 个浮点数。
    • 当序列长度 $L$ 较短时，这部分显存可以忽略。
    • 当序列长度 $L$ 很长（如长文本、高分辨率图像）时，这部分显存开销会变得明显。这也是为什么 FlashAttention 等优化技术如此重要，它们通过分块计算避免了完整存储这个 $L \times L$ 矩阵。

4. 总结对比表

特性 单头注意力 (SHA) 多头注意力 (MHA) 对比结论
前提 $d_{head} = d_{model}$ $h \times d_{head} = d_{model}$ 保持总容量一致
参数量 $4d_{model}^2$ $4d_{model}^2$ 相同
理论 FLOPs $O(L^2 \cdot d_{model})$ $O(L^2 \cdot d_{model})$ 相同
中间显存 1 个 $L \times L$ 注意力图 $h$ 个 $L \times L$ 注意力图 MHA 多占 $(h-1)L^2$
工程开销 无切分/拼接开销 有切分/转置/拼接开销 MHA 有微小开销
并行效率 单个大矩阵乘 多个小矩阵乘并行 MHA 通常更优或持平
表达能力 单一视角 多视角子空间 MHA 显著更强

3. 头部变体：MQA 与 GQA

为降低推理成本，常见两种变体：

  1. MQA：所有 Query 头共享同一组 Key/Value，极省 KV 开销。MQA 提出来自于 Meta 的 LLaMA 2，在长上下文中表现不错，但在短文本任务上可能略逊于 MHA。
  2. GQA：多个 Query 头共享一组 Key/Value，在质量与成本间折中。

详细拆解见：

4. 位置编码：为什么常用 RoPE

注意力机制本身不感知顺序，所以需要位置编码。RoPE 通过旋转变换把位置信息注入向量，在长上下文中表现通常更稳。

详细拆解见：

  • RoPE（旋转位置编码）理解与应用

5. 实现细节：QKV Bias

QKV 线性层中的 bias 看起来很小，但会影响训练动态与收敛行为，是否启用应通过任务实验决定。

详细拆解见：

  • QKV Bias 在 Transformer 中的作用

6. 推理侧重点：KV Cache

自回归推理时，历史 token 的 K/V 会缓存起来避免重复计算。上下文变长后，KV Cache 往往成为显存与吞吐瓶颈，这也是 MQA/GQA 受欢迎的重要原因。

评估时建议同时看：

  • 吞吐（tokens/s）
  • 延迟（首 token 与平均 token）
  • 长上下文稳定性

7. 工程加速：FlashAttention（概念）

FlashAttention 的核心价值是优化注意力计算的访存路径，减少不必要的内存读写，从而提升速度并降低显存压力。

它本质是“算子工程优化”，不改变注意力机制定义，但显著改变实际运行效率。

8. 怎么读 Attention 相关论文

我的建议是固定三问：

  1. 这项改动改善了什么瓶颈（质量、显存、延迟、吞吐）？
  2. 代价是什么（训练复杂度、部署复杂度、稳定性风险）？
  3. 在我的目标任务和预算下是否划算？

小结

Attention 不是单一技术点，而是一组“机制 + 工程”组合。先用总览建立坐标，再按专题深入，学习和落地都会更快。
