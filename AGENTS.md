# AGENTS.md

## 我如何辅助你工作

我会按这个方式协作：

1. 先确认你的目标，再直接动手实现。
2. 改动前说明会改什么，改动后给你可验证结果。
3. 优先做可维护方案，避免只改临时产物。
4. 需要你决定的地方，会给出简洁选项。

## 在这个仓库里我主要能做什么

1. 写博客：起草、改写、结构化论文阅读笔记。
2. 前端优化：视觉风格、动效、交互、移动端适配。
3. 站点维护：修复页面问题、整理目录和工作流。
4. 发布支持：提交、推送、排查构建与缓存问题。

## 文章编辑规范（重要）

请优先编辑 `post` 源文件，不要直接改生成后的 html。

- 源文件目录：`blog-source/source/_posts/*.md`
- 示例：`blog-source/source/_posts/qwen2-5-report-reading.md`

常用命令（在仓库根目录）：

```bash
scripts/post.sh list
scripts/post.sh new "文章标题"
scripts/post.sh build
scripts/post.sh server
```

## 你可以这样给我任务

1. `帮我写一篇关于 XXX 的论文阅读，风格偏技术，1500 字。`
2. `把首页标签改成可筛选文章，并保留炫酷动效。`
3. `把这次改动提交并推送。`
4. `检查这篇文章的结构和表达，给我直接可用的修改稿。`

## 默认交付内容

我每次会给你：

1. 改了哪些文件。
2. 做了什么关键变更。
3. 如何验证是否生效。
4. 是否已提交/推送。
